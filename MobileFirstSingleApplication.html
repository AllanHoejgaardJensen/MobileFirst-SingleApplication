Mobile First -> Single Applikation

det handler om at gå hårdt efter fuld funktionalitet på smartphones, og det skal give en god og genkendelig oplevelse på alle dimser, dvs. lille visuelt område - betjent med en tommelfinger

få det til at fungere på tværs af dimser og platforme...

Mobile First - Single Application - Cross platform

-> 1 applikation på tværs af af alle dimser

Fordele

Man skal komme helt ind til kernen af hvad der er nødvendigt at levere til brugerne for at de føler sig "ordentligt" betjent, de kan skifte til den dims der passer dem og stadigt overføre betjeningsintuiteten mellem dimser.

Bruger/Kunde vælger dims/device

-> til interaktionen med os

Mobile First
  • Der skal være noget interessant umiddelbart.
  • Det første man ser, skal kunne fortsættes
    • funktionalitet skal være lige ved “tomlen” - man vil typisk noget f.eks. checke saldo, se sidste posteringer, er der gået løn ind, betale... hvis det er banken der er tale om ...
    • Organiseringen skal være ordnet efter kendt behov.... og enkel
Single Application
  • man kan tilføje convinience links på dimser med større visuelt område...
  • funktionaliteten er den samme som for smartphone...
  • man kan tilføje tastatur og pointer support således tablets, desktop, superbrugere, skærmlæsere, *SE’ere etc.
*SE: Search Engines
Brugerne / Kunderne
  • vælger frit den dims de vil og kunne kommunikere med virksomheden på...tablets, desktop, superbrugere, skærmlæsere, *SE’ere etc.
  • får en genkendelig og fornuftig måde for de typer af applikationer hvor funktionaliteten er i centrum
  • anvender devices med touch, tastatur og pointer efter eget valg
